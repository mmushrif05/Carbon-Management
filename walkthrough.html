AI Engine 1
BOQ / Tender Parser
Upload a Bill of Quantities (PDF, Excel, or CSV). The AI extracts material quantities, maps each line to an ICE DB emission factor, and builds a carbon forecast scenario ‚Äî before any procurement order is placed.
üìã
What it does: Reads raw BOQ documents, extracts quantities, fuzzy-matches materials to the ICE database, converts units, and outputs a full carbon forecast with scenario comparison. Weeks of manual work ‚Üí seconds.

AI Engine 2
Document Intelligence ‚Äî RAG
Upload any project document. The AI analyses it across 5 dimensions ‚Äî material compliance, reduction opportunities, cross-document consistency, certification gaps, and actionable recommendations ‚Äî with every finding cited to the exact source page.
üîç
RAG pipeline: Documents are chunked, embedded, and stored. At query time, the most relevant chunks are retrieved and injected with live project carbon context ‚Äî so findings reference both the document and real project numbers simultaneously.

AI Engine 3 ‚Äî Real Integration Proof
Carbon Advisor ‚Äî AI reduction strategy
The Carbon Advisor is a live AI integration inside every project. It reads the actual project carbon data ‚Äî totals, materials, contractor performance ‚Äî and generates a specific, prioritised reduction strategy. This is not a static report. It is a real call to Claude AI with live project data as context.
üß†
How it works: On click, the platform builds a structured payload ‚Äî total baseline, actual, materials breakdown, contractor gap analysis ‚Äî and sends it to the AI. The response is rendered live in the project modal under the AI Advisor tab.
